`@unocss/preset-mini` is the foundational preset for UnoCSS. It includes only the most essential utility rules aligned with CSS properties, without opinionated or complex utilities like `container`, `animation`, or `gradient`.

This preset is a good starting point for building your own custom preset on top of familiar utilities from Tailwind CSS or Windi CSS.

## When to use preset-mini vs preset-wind3

Use `preset-mini` when you:

- Are building a custom preset and want a lean base to extend
- Only need basic CSS-property utilities (spacing, colors, typography, borders)
- Want to avoid pulling in animation, container, or gradient utilities

Use [`preset-wind3`](/presets/wind) when you:

- Want full Tailwind CSS v3 or Windi CSS compatibility
- Need utilities like `animate-*`, `container`, or gradient helpers
- Are migrating an existing Tailwind or Windi CSS project

<Steps>
  <Step title="Install the package">
    <CodeGroup>
      ```bash pnpm
      pnpm add -D @unocss/preset-mini
      ```
      ```bash yarn
      yarn add -D @unocss/preset-mini
      ```
      ```bash npm
      npm install -D @unocss/preset-mini
      ```
      ```bash bun
      bun add -D @unocss/preset-mini
      ```
    </CodeGroup>
  </Step>
  <Step title="Add to your config">
    ```ts
    import presetMini from '@unocss/preset-mini'
    import { defineConfig } from 'unocss'

    export default defineConfig({
      presets: [
        presetMini(),
        // ...other presets
      ],
    })
    ```
  </Step>
</Steps>

<Tip>
This preset is included in the `unocss` package — you can import it from there directly:

```ts
import { presetMini } from 'unocss'
```
</Tip>

## Features

### Dark mode

By default, this preset generates class-based dark mode with the `dark:` variant:

```html
<div class="dark:bg-red:10" />
```

This generates:

```css
.dark .dark\:bg-red\:10 {
  background-color: rgb(248 113 113 / 0.1);
}
```

#### Media query dark mode

To switch to `prefers-color-scheme` globally:

```ts
presetMini({
  dark: 'media'
})
```

The same class now generates:

```css
@media (prefers-color-scheme: dark) {
  .dark\:bg-red\:10 {
    background-color: rgb(248 113 113 / 0.1);
  }
}
```

### CSS `@layer`

Native CSS `@layer` is supported via the `layer-{name}:` variant:

```html
<div class="layer-foo:p4" />
<div class="layer-bar:m4" />
```

Generates:

```css
@layer foo {
  .layer-foo\:p4 {
    padding: 1rem;
  }
}
@layer bar {
  .layer-bar\:m4 {
    margin: 1rem;
  }
}
```

### Theme customization

Extend or override the default theme in your config. UnoCSS will deeply merge your values with the defaults:

```ts
presetMini({
  theme: {
    colors: {
      veryCool: '#0000ff',        // class="text-very-cool"
      brand: {
        primary: 'hsl(var(--hue, 217) 78% 51%)', // class="bg-brand-primary"
      }
    },
  }
})
```

<Warning>
The `breakpoints` property is not deeply merged — it is fully overridden when you provide custom values. See the [theme configuration docs](/config/theme#breakpoints) for details.
</Warning>

## Options

### `dark`

- **Type:** `'class' | 'media' | DarkModeSelectors`
- **Default:** `'class'`

Dark mode strategy. Use `'media'` for `prefers-color-scheme`, or pass a `DarkModeSelectors` object for custom selectors:

```ts
interface DarkModeSelectors {
  light?: string  // default: '.light'
  dark?: string   // default: '.dark'
}
```

### `attributifyPseudo`

- **Type:** `boolean`
- **Default:** `false`

Generate pseudo selectors as `[group=""]` instead of `.group`.

### `variablePrefix`

- **Type:** `string`
- **Default:** `'un-'`

Prefix for CSS custom properties generated by UnoCSS.

### `prefix`

- **Type:** `string | string[]`
- **Default:** `undefined`

Optional prefix for all utility class names.

### `preflight`

- **Type:** `boolean | 'on-demand'`
- **Default:** `true`

Controls CSS preflight generation:

- `true` — always generate preflight styles
- `false` — no preflight
- `'on-demand'` — only generate preflight for utilities that are actually used
